# 台灣好眠 App — 產品討論完整紀錄

---

## 一、起點：最初的 Idea

**團隊提出的構想：** 開發一款「解鎖台灣白噪音」的助眠 App

### 初步分析的優點

- 「台灣在地聲景」有差異化切入點
- 結合文化認同感和功能性需求
- 實地錄音內容難以複製

### 需要釐清的問題

- 「解鎖」機制的目的是什麼？
- 「白噪音」定位是否精確？（其實更接近環境音/soundscape）
- 使用場景要多廣？（純睡眠？還是包含專注、冥想？）

---

## 二、競品分析

### 國際主要競品

| App | 特色 | 市場地位 |
|-----|------|---------|
| Calm | 睡眠故事、名人配音、冥想課程 | 市值超過 20 億美元 |
| Headspace | 科學化冥想、與 Netflix 合作 | 6000 萬+ 下載 |
| Endel | AI 即時生成音景 | Apple Watch 年度最佳 |
| 潮汐 Tide | 極簡設計、番茄鐘結合 | 3000 萬用戶 |

### 台灣相關資源（非 App）

- 台灣自然聲音地圖
- 小島聲集所（商用音效庫）
- Freesound 上的台灣環境音

**關鍵發現：** 目前沒有專做「台灣在地聲景 + 睡眠」的 App，但單純的聲音沒有護城河。

---

## 三、釐清產品定位

經過討論，確認三個方向：

| 項目 | 確認的方向 |
|------|-----------|
| 核心定位 | 助眠工具（不是單純的白噪音播放器） |
| 資源整合 | 與健康 2.0 結合，提供專家內容與節目置入 |
| 黏著度機制 | 遊戲化解鎖，增加用戶持續使用的動力 |

## 四、目標受眾分析

| 優先級 | 用戶群 | 痛點 |
|--------|--------|------|
| P0 | 25-45 歲都市上班族 | 壓力大、入睡困難 |
| P1 | 健康 2.0 既有觀眾 | 信任節目，想要健康工具 |
| P2 | 海外台灣人 | 思鄉、想聽熟悉的聲音 |

---

## 五、用戶動機分析（5 Whys）

層層追問用戶為什麼會用這款 App：

| 層次 | 問題 | 答案 |
|------|------|------|
| Why 1 | 為什麼下載？ | 睡不好，想找方法改善 |
| Why 2 | 為什麼選這款？ | 健康 2.0 專家背書，感覺更專業可信 |
| Why 3 | 為什麼專家背書重要？ | 對睡眠問題焦慮，需要被「告訴該怎麼做」 |
| Why 4 | 為什麼被照顧感能讓他們留下？ | 改善睡眠需要時間，需要陪伴感和成就感 |
| Why 5 | 為什麼台灣聲景能創造連結？ | 這是「屬於我們的聲音」，有情感認同 |

**核心洞察：** 用戶下載是因為「睡不好」，但讓他們留下來的是「專家的安心感」和「收集台灣聲音的成就感」。

## 六、嚴厲檢視：可能失敗的 7 個理由

| 問題 | 風險 |
|------|------|
| 1. 市場已經飽和 | Calm、潮汐已經做到極致 |
| 2. 台灣聲景市場太小 | 天花板只有 2500 萬人 |
| 3. 健康 2.0 觀眾年齡偏高 | 40-65 歲不一定習慣用 App |
| 4. 遊戲化可能適得其反 | 睡前不該讓用戶有「收集壓力」 |
| 5. 內容製作成本高但護城河低 | 聲音本身容易被複製 |
| 6. 免費替代品太多 | YouTube、Freesound 都有免費的 |
| 7. TVBS 資源不一定持續投入 | 做完可能就沒人管了 |

---

## 七、解決方案

針對每個問題提出對策：

| 問題 | 解決方案 |
|------|---------|
| 市場飽和 | 打側翼戰，定位成「健康工具」而非「冥想 App」 |
| 市場太小 | 台灣聲景是鉤子，專家處方才是核心 |
| 觀眾年齡層 | 雙軌策略，同時服務長輩（台語引導、大字體）和年輕人 |
| 遊戲化反效果 | 改成「正向回饋」，通知放早上而非睡前 |
| 護城河低 | 護城河是「專家 + 數據 + 節目連動」的組合 |
| 免費替代品多 | 免費給聲音，付費給解決方案 |
| 資源不持續 | 設計低維護成本的更新機制（配合節目錄影順便錄內容） |

---

## 八、關鍵轉折：從「白噪音 App」到「睡眠教練」

### 核心洞察

> 白噪音只是「工具」，不是「解決方案」。
> 用戶缺的不是雨聲，是「有人幫我搞定睡眠這件事」。

### 定位轉變

| 原本思維 | 新思維 |
|---------|--------|
| 提供 50 種聲音讓你選 | 告訴你「今晚聽這個」 |
| 用戶自己摸索 | 系統引導你一步步改善 |
| 打開 → 播放 → 關掉 | 評估 → 處方 → 追蹤 → 調整 |
| 賣的是「內容」 | 賣的是「結果」 |

### 新的核心體驗流程

```
第一次使用
───────────
睡眠評估 → 問題診斷 → 專屬處方
(5題問卷)   (你是這種類型)   (14天計畫)

每天晚上
───────────
今日任務 → 引導練習 → 聲景播放
(教練說)    (跟著做)    (入睡)

每天早上
───────────
睡眠回報 → 數據追蹤 → 計畫調整
(睡得如何)  (進步了嗎)  (明天換這個)
```

## 九、台灣地圖的角色調整

確認台灣地圖聲景功能保留，但角色改變：

| 原本設計 | 調整後設計 |
|---------|-----------|
| 地圖是首頁，收集是核心玩法 | 睡眠計畫是首頁，地圖是「探索」分頁 |
| 用戶為了「收集」而打開 | 用戶為了「今晚的任務」而打開 |
| 解鎖靠「遊戲化條件」 | 解鎖靠「完成睡眠計畫」 |

### 新的 App 架構

| Tab | 名稱 | 功能 |
|-----|------|------|
| Tab 1 | 今晚 | 今日任務、處方推薦、專家引導（主頁） |
| Tab 2 | 探索 | 台灣地圖、聲景收集、解鎖進度（彩蛋） |
| Tab 3 | 我的 | 好眠旅程、累積成就、進步紀錄 |

## 十、數據追蹤的焦慮問題

### 發現的問題

睡不好的人看到自己的睡眠數據，可能會更焦慮（Orthosomnia 正睡症）。

### 設計原則

| 原則 | 做法 |
|------|------|
| 追蹤行為，不追蹤結果 | 記錄「你做了什麼」，不記錄「你睡幾小時」 |
| 強調進步，不強調現況 | 「比上週進步」而不是「目前分數」 |
| 給選擇權 | 想看詳細數據的人可以自己開，預設關閉 |

### 語氣調整

| 避免說 | 改成說 |
|--------|--------|
| 「睡眠品質差」 | 「沒關係，今晚再試試」 |
| 「你已經 3 天沒使用」 | 「歡迎回來，今晚一起好好睡」 |
| 「睡眠時數減少」 | （不顯示負面數據） |

## 十一、睡眠問卷設計

### 5 題快速問卷

| 題目 | 目的 |
|------|------|
| Q1. 躺上床後多久能睡著？ | 判斷入睡困難程度 |
| Q2. 晚上會醒來嗎？ | 判斷淺眠/中斷問題 |
| Q3. 早上起床的感覺？ | 判斷睡眠品質 |
| Q4. 睡前腦袋的狀態？ | 判斷焦慮程度 |
| Q5. 影響睡眠最大的因素？ | 決定處方方向 |

### 四種睡眠類型

| 類型 | 特徵 | 推薦處方方向 |
|------|------|-------------|
| 入睡困難型 | 躺很久睡不著 | 呼吸法 + 漸進放鬆 |
| 淺眠易醒型 | 半夜一直醒 | 持續穩定的環境音 |
| 焦慮思緒型 | 腦袋轉不停 | 正念引導 + 緩慢節奏聲景 |
| 環境敏感型 | 太吵或太安靜 | 遮蔽型白噪音 |

## 十三、Demo 展示流程

展示給團隊/主管時的流程：

1. **開場**：「這是一款睡眠教練 App，不只是白噪音」
2. **睡眠評估**：填問卷 → 顯示「你是入睡困難型」
3. **處方推薦**：系統推薦 7 天計畫 + 今晚的任務
4. **執行任務**：播放呼吸引導 → 播放聲景
5. **彩蛋展示**：切到地圖 Tab，「完成計畫就能解鎖更多台灣聲景」
6. **收尾**：「這是健康 2.0 醫師團隊設計的睡眠解決方案」

## 十四、最終產品定位總結

| 項目 | 內容 |
|------|------|
| **一句話定位** | 結合健康 2.0 專家的睡眠教練 App，用台灣在地聲景陪你好好睡 |
| **核心價值** | 不只給你聲音，給你「解決方案」 |
| **差異化優勢** | 專家背書 + 睡眠處方 + 台灣聲景 + 節目連動 |
| **主要功能** | 睡眠評估 → 個人化計畫 → 每日任務 → 正向回饋 |
| **次要功能** | 台灣聲景圖鑑（作為獎勵和彩蛋） |
| **目標用戶** | 25-45 歲有睡眠困擾的都市上班族 + 健康 2.0 觀眾 |
