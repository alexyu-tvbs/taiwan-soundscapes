# 產品需求文件 — Taiwan Soundscapes（好眠秘境）

**作者：** Alex Yu **日期：** 2026-01-29

## 摘要

Taiwan Soundscapes（好眠秘境）是一個為內部「好眠服務 Prototype 提案大賽」打造的 Web Prototype。它將白噪音重新想像為「台灣聲音旅行」——以互動式台灣地圖取代傳統的播放清單介面，讓使用者探索真實地點並聆聽當地聲景。

**核心概念：** 一個互動式 SVG 台灣地圖，發光的地點播放獨特的聲景音訊，搭配現場攝影。鎖定的地點展示 Gamification 機制——使用者透過維持良好的睡眠習慣來解鎖隱藏的聲景。設計刻意反轉業界的焦慮模式：沒有倒數計時、沒有懲罰，只有溫暖的鼓勵。

**Phase 2 擴展：** 在已完成的地圖 Prototype 基礎上，Phase 2 將產品從「台灣聲音地圖」提升為「睡眠教練概念」。睡眠評估 Onboarding 流程、個人化的「今晚」首頁（含睡眠處方）、追蹤進度的「我的旅程」Tab，以及「產品故事」頁面，層層疊加在既有的地圖體驗之上。這將 Demo 敘事從展示單一功能，轉變為展示一個具備清晰產品思維的完整睡眠方案。所有新增內容皆為靜態前端——不需要 Backend 或資料持久化。

**目標：** 被評審 Gaod 選入 Top 3，進入 IP 提案。Prototype 本身就是最好的 Pitch——不需要額外解釋。

**限制：** 1 人（Alex Yu）+ AI 工具、純 Web、免費 / Placeholder 素材。

## 使用者旅程

### 旅程 1：Demo 體驗者 — 小薰（同事 / 觀眾）

**場景：** 小薰是一位坐在會議室裡觀看各參賽者 Prototype Demo 的同事。看了幾個提案後，輪到 Alex。

**開場：**
小薰在筆電上打開連結，看到一張台灣地圖浮現在深色背景上，幾個地點柔和地發著光。標題寫著「好眠秘境 — 用耳朵旅行台灣」。她心想：「這看起來跟一般的白噪音 App 不一樣。」

**鋪陳：**
她點擊發光的「淡水河夕陽」地點。畫面切換到一張河岸夕陽照片，耳機裡傳來輕柔的水聲和遠處的船笛聲。她彷彿被傳送到了淡水河畔。接著她點擊「阿里山雲海」——蟲鳴和清晨森林的聲音伴隨著雲海照片響起。

**高潮：**
她注意到地圖上有些地點是暗淡的，帶有鎖頭圖示。好奇之下，她點擊「蘭嶼飛魚季」。一個溫柔的提示出現：「連續好眠 14 天，解鎖這片海洋」。她恍然大悟——這不只是播放器；這是一個「好好睡覺就能收集台灣聲音」的系統。

**結局：**
Demo 結束後，小薰自己重新打開網頁，聆聽每一個發光的地點。她在群組裡傳訊息：「我最喜歡 Alex 的——那個台灣地圖太美了。」

### 旅程 2：評審 Gaod — 評估視角

**場景：** Gaod 今天正在評估所有參賽者的 Prototype，逐一打分。他重視創意、視覺呈現和商業潛力。他已經看過幾個不錯的提案。

**開場：**
Gaod 打開 Alex 的 Prototype 連結。一張 Dark Theme 的台灣地圖出現，地點標記著柔和的發光圓點。第一印象：「視覺品質扎實——這看起來跟其他人不一樣。」

**鋪陳：**
他點擊一個發光的地點；聲景開始播放，搭配一張場景照片。他點擊另一個——不同的聲音，不同的風景。概念在他腦中成形：「這是用台灣在地的聲音當白噪音。」他注意到有些地點是鎖定的；點擊後顯示解鎖條件。Gamification 機制變得清晰。

**高潮：**
Gaod 進行腦內評估：「創意——台灣地圖加聲音旅行，這個概念有記憶點。視覺——地圖的呈現品質超越一般 Prototype 水準。商業——台灣在地內容是國際品牌無法複製的護城河，解鎖機制驅動使用者留存。」

**結局：**
他將 Taiwan Soundscapes 放入 Top 3 候選名單。在最終篩選時，他寫下：「概念清晰、視覺精緻、商業邏輯完整。建議進入 IP 提案。」

### 旅程 3（Phase 2）：睡眠教練 Demo — 完整敘事流程

**場景：** 同樣的比賽 Demo，但現在 Prototype 說了一個更豐富的故事。Alex 向 Gaod 和觀眾展示擴展後的版本。

**第一幕 —「這個 App 懂我」：**
Alex 打開 Prototype。畫面並非直接跳到地圖，而是出現一個睡眠評估：5 個關於睡眠習慣的簡單問題。他填完。結果頁面出現：「你是**入睡困難型** — 你的思緒在睡前會不斷奔跑，難以靜下來。」下方寫著：「我們為你準備了 7 天急救方案。」他點擊「開始我的計畫」。觀眾探頭——這不是聲音播放器。

**第二幕 —「今晚的處方」：**
「今晚」Tab 作為首頁出現。進度條顯示「入睡困難急救包 — 第 5 天 / 共 7 天」。下方兩張處方卡片：一個呼吸練習（「4-7-8 呼吸法 · 3 分鐘 · 江醫師引導」）和一個聲景推薦（「淡水河夕陽 · 舒緩水聲」）。Alex 點擊聲景卡片——App 導航到探索 Tab，開始在地圖上播放淡水聲景。過渡感覺無縫：教練開了處方，地圖來執行。

**第三幕 —「我在進步」：**
Alex 切換到「我的」Tab。累積數據出現：12 次完成的練習、連續 5 天、已解鎖 5 個聲景。一則溫暖的訊息寫著：「你連續 3 天在 11 點前開始準備睡覺——這是很棒的習慣養成！」觀眾理解了：這個 App 追蹤的是行為，不是睡眠時數。沒有焦慮，只有鼓勵。

**第四幕 —「他們想得很深」：**
Alex 點擊「產品故事」連結。一個設計精美的頁面展開——產品願景、競品定位、目標客群和設計哲學。它解釋了為什麼這是「睡眠教練」而非「白噪音播放器」，健康 2.0 專家整合如何建立護城河，以及為什麼反焦慮設計很重要。Gaod 心想：「他們不只是做了一個 Prototype——他們做了產品思考。」

**結局：**
Gaod 的腦內計分卡更新：「Phase 1 展現了創意執行。Phase 2 展現了產品深度。這個團隊理解從 Prototype 到真實產品需要什麼。」

## 創新與新穎模式

### 識別出的創新領域

**1. 品類重新定義：從播放清單到聲音地圖**
現有的白噪音產品使用「選擇音訊檔案 → 按播放」的 UI 範式。Taiwan Soundscapes 將互動模型從「播放清單」翻轉為「地理探索」——地圖就是介面，點擊地點就是選擇曲目。這不是功能改進；這是品類重新定義。

**Prototype 展示：** 打開 App 映入眼簾的是台灣地圖（不是音訊檔案清單）；所有互動都圍繞著地圖進行。

**2. 在地文化 × 健康行為：不可複製的護城河**
將「台灣地理認同」與「睡眠習慣養成」綁定——好好睡覺就能收集台灣的聲音。這個組合在全球市場上沒有先例：Calm 做通用冥想、Tide 做自然聲音，但沒有人用「國家文化地圖」作為核心產品引擎。

**Prototype 展示：** 所有地點名稱都是真實的台灣目的地（阿里山、蘭嶼、太魯閣等），搭配在地場景攝影，讓評審立刻辨識出「這是台灣」。

**3. 反焦慮 Gamification：刻意反轉業界常規**
業界慣例使用分數、排行榜和倒數計時器來驅動行為——往往製造焦慮。Taiwan Soundscapes 刻意反轉：不顯示「還剩 X 天」、不懲罰中斷、驚喜是額外獎勵而非必要條件。

**Prototype 展示：** 鎖定地點的提示使用溫暖、正面的語言（「連續好眠 14 天，解鎖這片海洋」），而非倒數或警告。

### 驗證方式

比賽本身就是第一輪驗證：
- 評審是否在 60 秒內理解「地圖 = 介面」的品類重新定義？
- 評審是否感受到「台灣在地」的情感連結與商業護城河？
- 溫和的解鎖機制語言是否讓評審覺得「這跟其他 App 不一樣」？

### Audio 素材策略

Prototype 的 Audio 來源可採用彈性方式：
- **YouTube 嵌入**（透過 IFrame API 搭配隱藏播放器 UI）使用現有的台灣自然聲音影片

### Phase 2 — 睡眠教練概念驗證

**目標：** 將 Prototype 從「台灣聲音地圖」提升為「結合台灣聲景的睡眠教練」。所有新增皆為靜態前端——無 Backend、無資料持久化。目的是強化比賽 Demo 的敘事。

**限制：** 仍為靜態 SPA。問卷結果與進度資料不會持久化（頁面重新整理即重置狀態）。分支邏輯為簡單的前端條件判斷，非推薦引擎。

#### P2-1：底部 Tab 導航

| Tab | 標籤 | 內容 |
|-----|------|------|
| Tab 1 | 今晚 (Tonight) | 睡眠處方首頁（新增，Onboarding 後的預設頁） |
| Tab 2 | 探索 (Explore) | 既有台灣聲景地圖（從全螢幕移入） |
| Tab 3 | 我的 (My Journey) | 進度追蹤與成就（新增） |

- Active Tab 有視覺指標
- Tab Bar 在三個視圖間持續可見
- 首次造訪（Onboarding 完成前），Tab 不顯示；問卷為全螢幕

#### P2-2：睡眠評估問卷（Onboarding）

首次造訪時呈現 5 題問卷：

| # | 問題 | 目的 |
|---|------|------|
| Q1 | 躺上床後多久能睡著？ | 評估入睡困難程度 |
| Q2 | 晚上會醒來嗎？ | 評估睡眠連續性 |
| Q3 | 早上起床的感覺？ | 評估睡眠品質 |
| Q4 | 睡前腦袋的狀態？ | 評估睡前焦慮 |
| Q5 | 影響睡眠最大的因素？ | 決定處方方向 |

每題有 3-4 個選擇題選項。UI 為每螢幕一題，搭配進度指標。

**分支邏輯：** 簡單的優先級判斷——「最嚴重」的回答決定類型。無複雜評分模型。

#### P2-3：睡眠類型結果頁

3 種可能結果：

| 類型 | 觸發條件 | 顯示內容 |
|------|----------|----------|
| 入睡困難型 | Q1 回答顯示 > 30 分鐘 | 呼吸練習 + 慢節奏聲景 |
| 淺眠易醒型 | Q2 回答顯示頻繁醒來 | 連續穩定的環境音 |
| 焦慮思緒型 | Q4 回答顯示思緒紛飛 | 正念引導 + 自然聲景 |

結果頁顯示：類型名稱、簡短描述、建議方式，以及「開始我的計畫」CTA 按鈕。

#### P2-4：「今晚」首頁

Onboarding 後作為 Tab 1 顯示的靜態處方頁面：

- **計畫進度條：** 「入睡困難急救包 — 第 5 天 / 共 7 天」（Hardcoded）
- **呼吸練習卡片：** 僅展示用卡片，顯示「4-7-8 呼吸法 · 3 分鐘 · 江醫師引導」。點擊顯示靜態資訊 Overlay（無互動動畫）。
- **聲景推薦卡片：** 顯示地圖上的推薦地點。點擊導航至探索 Tab，自動選擇該地點播放。
- **教練建議：** 底部一行提示（例如「今天試著比昨天早 15 分鐘上床」）

處方內容依睡眠類型結果變化（3 種卡片內容變體），但版面配置相同。

#### P2-5：「我的旅程」頁面

作為 Tab 3 的靜態成就展示：

- 累積數據（Hardcoded）：完成的練習次數、最長連續天數、已解鎖聲景數
- 正向回饋訊息（例如「你連續 3 天在 11 點前開始準備睡覺，這是很棒的習慣！」）
- 當前計畫進度（與「今晚」的進度條一致）

#### P2-6：既有地圖整合

- 既有的全螢幕地圖移入 Tab 2（「探索」）
- 新增地圖上方的收集進度指標：「已收集 3/10 個台灣聲景」
- 新增連結回睡眠計畫的提示：「完成「入睡困難急救包」即可解鎖：台東稻田」
- 所有既有地圖功能（地點圓點、Audio 播放、鎖定 Overlay）維持不變

#### P2-7：產品故事頁

一個專屬頁面（透過 App 內的連結 / 按鈕存取——例如 Header 中的資訊圖示或「我的」中的區塊），呈現產品的概念與設計思維。內容取自產品討論文件：

- **產品願景：** 從「白噪音播放器」到「睡眠教練」——為什麼這個產品存在
- **競品分析：** 如何定位在 Calm、Headspace、Tide、Endel 之間——以及它們都沒做到的事
- **目標客群：** 這是為誰打造的，以及 5 Whys 分析的核心洞察
- **設計哲學：** 反焦慮 Gamification、正向回饋、「追蹤行為而非結果」
- **差異化護城河：** 專家背書（健康 2.0）+ 台灣文化認同 + 節目整合
- **完整產品 Roadmap：** Prototype 之後的方向（評估 → 處方 → 追蹤 → 調整循環）

此頁面為 Presentation-Grade——乾淨的排版、結構化的段落，可能搭配簡單的插圖或圖示。它作為內嵌的 Pitch Deck，讓評審可以依自己的節奏探索。

#### Phase 2 — 明確排除的範圍

- 呼吸練習互動動畫（僅展示卡片）
- 跨頁面重新整理的資料持久化
- 真實的個人化引擎或推薦演算法
- 推播通知或時間觸發機制
- 使用者帳號或身份驗證
- 任何形式的 Backend 服務

## 功能需求

### 地圖探索

- **FR1：** 使用者可在頁面載入時看到互動式台灣地圖作為主要介面
- **FR2：** 使用者可看到地圖上的地點標記，以視覺區分為已解鎖（發光）或鎖定（暗淡）
- **FR3：** 使用者可點擊已解鎖的地點標記來選取並觸發其聲景
- **FR4：** 使用者可點擊鎖定的地點標記來查看其解鎖條件
- **FR5：** 使用者可在主介面上看到品牌標語「好眠秘境 — 用耳朵旅行台灣」

### 聲景播放

- **FR6：** 選取已解鎖地點時，使用者可聽到 Audio 播放
- **FR7：** 使用者可暫停和繼續當前播放的聲景
- **FR8：** 使用者可調整播放中聲景的音量
- **FR9：** 使用者可看到與當前播放地點相關的場景照片
- **FR10：** 使用者可在已解鎖地點之間切換，停止前一個聲景並播放新的

### 解鎖系統

- **FR11：** 使用者可在尚未解鎖的地點上看到鎖定指標
- **FR12：** 使用者可查看每個鎖定地點的具體解鎖條件（例如「連續好眠 14 天」）
- **FR13：** 使用者可看到以鼓勵語言呈現的解鎖條件，採用正向成就模式（例如「[正向行動] 來解鎖 [獎勵]」），無倒數計時器或懲罰框架

### 地點內容

- **FR14：** 使用者可看到 3 個已解鎖地點，含完整 Audio 與圖片：淡水河夕陽、阿里山雲海、基隆港浪

- **FR18：** 使用者可看到已解鎖地點標記上的 Glow 動畫效果
- **FR19：** 使用者可看到鎖定地點標記上暗淡 / 靜音的視覺狀態搭配鎖頭圖示

## 技術架構

### Tech Stack

- **架構：** SPA (Single Page Application)
- **Framework：** React + Vite
- **Styling：** Tailwind CSS
- **Animation：** Motion（前身為 Framer Motion）
- **地圖：** 自製 SVG 台灣地圖（非 Mapbox/Leaflet — 視覺衝擊優先於地理精確度）
- **Audio：** HTML5 Audio API（來源：免費音訊檔案 + 可選 YouTube 嵌入）
- **部署：** Vercel 或 Netlify（靜態 SPA）

### Component 結構

**Phase 1 Components（既有）：**

| Component | 職責 |
|-----------|------|
| `App` | Root Layout、Dark Theme、全域 Audio 狀態 |
| `TaiwanMap` | SVG 地圖渲染、地點圓點、Glow 效果 |
| `LocationDot` | 單一可點擊地點（已解鎖 / 鎖定狀態） |
| `SoundscapePlayer` | Audio 播放控制（播放 / 暫停 / 音量 / 淡出） |
| `LocationDetail` | 場景照片 + 地點名稱 + 描述 Overlay |
| `LockOverlay` | 鎖定地點提示，含解鎖條件訊息 |

**Phase 2 Components（新增）：**

| Component | 職責 |
|-----------|------|
| `TabBar` | 底部 Tab 導航，含 Active 狀態指標 |
| `SleepAssessment` | 5 題 Onboarding 流程，含進度指標 |
| `SleepTypeResult` | 結果頁，顯示睡眠類型 + 描述 + CTA |
| `TonightPage` | 處方首頁：進度條、卡片、教練建議 |
| `PrescriptionCard` | 呼吸練習或聲景推薦的單張卡片 |
| `MyJourneyPage` | 成就數據 + 回饋訊息 + 進度 |
| `ProductStory` | 可捲動頁面，呈現產品概念與願景 |

### State Management

- React useState / useContext 即足夠（不需要 Redux）
- Phase 1 狀態：當前播放地點、播放狀態、解鎖狀態
- Phase 2 狀態：Onboarding 完成旗標、判定的睡眠類型、Active Tab
- 不需要持久化（Prototype 重新整理即重置）
- 睡眠類型結果保存在記憶體中，驅動今晚頁面的內容變體

### 實作考量

- **SVG 地圖：** 使用或建立簡化的台灣輪廓 SVG，以絕對座標定位地點圓點
- **Audio：** 預載已解鎖地點的音訊檔案；鎖定地點不需載入 Audio
- **Dark Theme：** Tailwind `dark` class 作為預設，深海軍藍 / 炭灰背景
- **Glow 效果：** CSS `box-shadow` 或 SVG `filter` 實現地點圓點的 Glow 動畫
- **場景照片：** 來自 Unsplash/Pexels 的台灣地點免費圖庫照片

## 風險緩解

### Phase 1 風險

| 風險類型 | 風險 | 緩解方式 |
|----------|------|----------|
| **技術** | SVG 地圖互動不夠精緻，無法傳達「品類重新定義」 | 優先處理地圖視覺——這是第一印象 |
| **技術** | Audio 播放在不同瀏覽器間不一致 | 預先測試 Chrome + Safari；指定 Demo 瀏覽器 |
| **內容** | Placeholder Audio 來源降低沉浸感 | 選擇有氛圍感的免費音源；搭配高品質場景攝影來補償 |
| **市場** | 評審無法理解概念 | 品牌標語 + 自主引導 UI 讓概念自己說話；鎖定地點提示文案必須清晰 |
| **資源** | 一週不夠完成所有功能 | 鎖定地點零 Audio 成本；先建核心 3 個聲景 + 地圖，再打磨 |

### Phase 2 風險

| 風險類型 | 風險 | 緩解方式 |
|----------|------|----------|
| **範圍** | Phase 2 新增分散了核心地圖體驗的焦點 | Tab 導航將地圖保留為專屬 Tab；Demo 時可直接跳到任何 Tab 跳過 Onboarding |
| **UX** | Onboarding 問卷在時間受限的 Demo 中感覺太慢 | 維持 5 題，每螢幕一題，快速點選 UI；整個流程應在 30 秒內完成 |
| **敘事** | 「今晚」頁面使用 Hardcoded 資料感覺不真實 | 使用合理的日期和進度值；Demo 的重點是傳達概念，不是證明真實功能 |
| **內容** | 產品故事頁內容對評審來說太長或太密 | 以清晰的標題結構化，每段保持 2-3 句，使用視覺層級讓掃讀更容易 |
| **技術** | 新增 Tab 和頁面導致既有地圖的 Layout Regression | 將既有地圖包裝在探索 Tab Container 中，不修改內部地圖邏輯；測試 Tab 間的轉場 |
| **設計** | Phase 2 新頁面與 Phase 1 既有視覺打磨之間的一致性 | 重用既有的 Dark Theme、配色、排版和 Motion 動畫模式 |
