# 1. 台灣秘境白噪音 Taiwan Soundscapes

## 1.1 Idea 發想檢核

| 檢核項目                 | 內容                                                              |
| ------------------------ | ----------------------------------------------------------------- |
| **對象是誰（Who）**      | 20-50 歲台灣用戶，喜歡收集成就、對台灣有情感認同者                |
| **想解決的問題（What）** | 國際睡眠 App 的白噪音都是國外場景，缺乏親切感與歸屬感             |
| **如何解決（How）**      | 用台灣在地聲音 + 解鎖收藏機制，創造情感連結與持續使用動力         |
| **競品考量**             | Calm/Headspace 無台灣內容；潮汐有自然聲但非台灣特色；目前市場空白 |
| **獨特性**               | 唯一以「台灣秘境」為主題的睡眠音景，結合旅遊情感與睡眠需求        |
| **健康 2.0 USP**         | 有外景拍攝團隊與經驗，可實地錄製高品質在地音景                    |

---

## 1.2 功能概述

> **一句話定義：** 用好眠表現解鎖台灣各地的自然聲音，收藏專屬於你的台灣聲音地圖。

### 核心價值主張

- **情感連結**：聽著熟悉的台灣聲音入睡，比陌生的異國聲音更有安全感
- **成就驅動**：解鎖機制創造持續使用動力，但不造成焦慮
- **文化認同**：「這是我們台灣的聲音」創造品牌差異化

---

## 1.3 目標用戶畫像

### 主要用戶：小琳（28 歲，台北上班族）

- **背景**：在台北工作，老家在台南，偶爾想念南部的海浪聲
- **痛點**：用過 Calm 但覺得國外的雨聲、森林聲太陌生
- **動機**：想要有親切感的聲音陪伴入睡
- **行為**：喜歡收集東西，會為了解鎖成就持續使用

### 次要用戶：阿明（45 歲，企業主管）

- **背景**：經常出差，懷念假日去山上的放鬆感
- **痛點**：出差時環境吵雜睡不好
- **動機**：想在任何地方都能聽到台灣的自然聲音
- **行為**：願意付費獲得好品質的體驗

# 2. KOL 晚安計畫 KOL Goodnight Program

## 2.1 Idea 發想檢核

| 檢核項目                 | 內容                                                    |
| ------------------------ | ------------------------------------------------------- |
| **對象是誰（Who）**      | 18-40 歲，有追蹤特定創作者習慣的用戶                    |
| **想解決的問題（What）** | 想聽熟悉的聲音入睡，但 YouTube/Podcast 廣告打斷體驗     |
| **如何解決（How）**      | 與台灣創作者合作製作獨家睡前內容，無廣告、專為入睡設計  |
| **競品考量**             | Calm 有 Matthew McConaughey 等名人，但無華語/台灣創作者 |
| **獨特性**               | 台灣創作者 × 獨家內容 × 粉絲經濟，三者結合              |
| **健康 2.0 USP**         | 多年藝人合作經驗，有現成的來賓資源與關係網絡            |

---

## 2.2 功能概述

> **一句話定義：** 你喜歡的創作者，專門為你錄製的睡前內容，只在好眠 App 聽得到。

### 核心價值主張

- **熟悉感**：聽熟悉的聲音入睡比陌生的更有安全感
- **獨家性**：「只有這裡有」創造下載與訂閱動力
- **導流效應**：創作者粉絲成為 App 用戶

---

## 2.3 內容類型規格

### 2.3.1 內容類型矩陣

| 類型          | 時長       | 風格               | 適合創作者              | 範例                     |
| ------------- | ---------- | ------------------ | ----------------------- | ------------------------ |
| **睡前故事**  | 15-30 分鐘 | 溫柔敘事、慢節奏   | 作家、配音員、Podcaster | 朗讀自己的作品、改編童話 |
| **ASMR 陪伴** | 20-40 分鐘 | 輕聲細語、音效觸發 | ASMR 創作者             | 耳語聊天、翻書聲、敲擊聲 |
| **冥想引導**  | 10-20 分鐘 | 平靜引導、呼吸指示 | 瑜伽老師、身心靈導師    | 身體掃描、放鬆冥想       |
| **晚安問候**  | 3-5 分鐘   | 溫暖對話、鼓勵     | 藝人、網紅              | 「今天辛苦了，晚安」     |
| **深夜電台**  | 30-45 分鐘 | 輕鬆聊天、音樂穿插 | Podcaster、DJ           | 讀聽眾來信、分享故事     |

### 2.3.2 內容製作規格

| 項目         | 規格                                        |
| ------------ | ------------------------------------------- |
| **錄音環境** | 專業錄音室或安靜居家環境（背景噪音 < 30dB） |
| **音質要求** | 48kHz/24bit，後製降噪、音量標準化           |
| **節奏要求** | 語速比正常慢 20%，句間停頓 2-3 秒           |
| **背景音樂** | 可選擇加入輕柔背景音（60bpm 以下）          |
| **禁止內容** | 刺激性話題、突然的大聲、恐怖元素            |

### 2.4.1 合作層級

| 層級           | 內容量               | 合作方式                   | 預估成本/月           |
| -------------- | -------------------- | -------------------------- | --------------------- |

### 2.4.2 分潤機制

```
收聽分潤公式：
每 1,000 次收聽 = NT$ 50（付費訂閱用戶收聽）
每 1,000 次收聽 = NT$ 10（免費用戶收聽）

範例：
某創作者內容月收聽 50,000 次（80% 付費、20% 免費）
= 40,000 × $0.05 + 10,000 × $0.01
= $2,000 + $100 = NT$ 2,100 分潤/月
```

# 3. 雙模態陪伴系統 Dual-Mode Companion

## 3.1 Idea 發想檢核

| 檢核項目                 | 內容                                                                |
| ------------------------ | ------------------------------------------------------------------- |
| **對象是誰（Who）**      | 各年齡層，需要被照顧感但不想被打擾的用戶                            |
| **想解決的問題（What）** | 想被關心但不想一直操作 App；有時需要主動提醒，有時只想安靜          |
| **如何解決（How）**      | Widget 被動陪伴（常駐存在）+ 語音主動介入（關鍵時刻），兩種模式切換 |
| **競品考量**             | 目前沒有 App 同時做到「被動存在」+「適時主動」的雙模態設計          |
| **獨特性**               | 像飯店管家一樣——平時不打擾，需要時立刻出現                          |
| **健康 2.0 USP**         | 品牌形象溫暖，適合「被照顧」的產品定位                              |

---

## 3.2 功能概述

> **一句話定義：** 不打擾的存在感 + 適時的溫暖介入，像一個懂你的朋友。

### 核心設計哲學

- **被動模式**：永遠在那裡，但不吵你
- **主動模式**：在你需要的時候，溫柔開口
- **不強迫**：所有介入都可以忽略，不會有罪惡感

---

## 3.3 模式一：Widget 被動陪伴

### 3.3.1 Widget 設計規格

| 項目         | 規格                                            |
| ------------ | ----------------------------------------------- |
| **尺寸**     | iOS：小/中/大三種；Android：2×2 / 4×2 / 4×4     |
| **更新頻率** | 每小時更新一次內容                              |
